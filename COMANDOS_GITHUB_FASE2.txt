# COMANDOS PARA ENVIAR O PROJETO (FASE 2) AO GITHUB

Siga estes passos exatamente no seu terminal (Git Bash ou PowerShell) dentro da pasta 'projeto-devops-fase2':

---

### Passo 1: Limpar e Configurar o Novo Repositório
```bash
git init
git remote remove origin  # Se der erro ignore, é só para garantir
git remote add origin https://github.com/airtonjunior-dev/projeto-devops-fase2.git
```

### Passo 2: Adicionar os Arquivos e Commitar
Como você não tem um arquivo README.md, use o `git add .` para pegar tudo (Dockerfile, RELATORIO_FINAL.md, etc.):
```bash
git add .
git commit -m "Entrega Fase 2 - DevOps na Pratica"
```

### Passo 3: Definir a Branch e Enviar
```bash
git branch -M master
git push -u origin master -f
```

---

### Como confirmar que deu certo?
1. Acesse o link: https://github.com/airtonjunior-dev/projeto-devops-fase2
2. Verifique se os arquivos aparecem lá.
3. Clique na aba **"Actions"** e veja se o "Pipeline CI/CD - Fase 2" começou a rodar!

Se o Actions ficar com um **check verde (✅)**, você está pronto para entregar! 🚀🏆
